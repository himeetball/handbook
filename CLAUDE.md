# Development Notes

## README Style
User prefers a playful, casual tone in documentation - not too formal or corporate. Include emojis where appropriate and write like you're explaining to a friend. Keep it approachable and fun while still being informative. Important: Use jokes, puns, and expressions sparingly - they lose their charm when overused.

## Multi-language Guidelines
- Use informal, personal "you" form in all languages (tu/tu/tu instead of formal vous/voi/vocês)
- Each language should use authentic cultural expressions and slang - not corporate translations
- Write like a native speaker talking casually to friends

### Portuguese (Portugal)
- Greeting: "Tá tudo?" (very casual Portuguese greeting)
- Use expressions like "cenas bueda fixes" (very cool stuff), "mesmo fixes" (really cool)
- "malta da equipa" instead of formal "nossa equipa"
- "bacano" (cool), "num instante/super rápido" instead of formal words
- "tá a andar de mota" (and you're all set!/and that's it done!), "tá feito" (it's done/sorted)

### Italian 
- Greeting: "Bella!" or "Ciao!"
- Use Roman/informal expressions: "roba figata" (cool stuff), "spacca" (rocks/awesome)
- "in un lampo" (in a flash) instead of formal "velocemente"

### French
- Greeting: "Salut!"
- Use casual expressions: "c'est ouf" (that's crazy cool), "des trucs de fou" (crazy cool stuff)
- "hyper vite" or "en deux secondes" instead of formal "rapidement"

### General Rules
- Always include (MeetBall Handbook) in parentheses after translated titles/names
- Keep it conversational like talking to a friend
- Each version should feel authentic to that culture, not a translation
