## 📝 **Usage Guidelines**

### **When to Use This Template**

- **Bug Reports**: Documenting issues found in the application
- **Feature Requests**: Proposing new functionality
- **Documentation**: Requesting documentation updates
- **Enhancements**: Improving existing features
- **Epics/Stories**: Breaking down large features

### **Best Practices**

- **Clear Description**: Provide clear, concise descriptions
- **Acceptance Criteria**: Define what "done" looks like
- **Proper Labels**: Use consistent labeling system
- **Attachments**: Include audio-visual when relevant
- **Related Issues**: Link related work

### **Success Criteria**

- Issue is clearly defined
- Acceptance criteria are specific and testable
- Proper labels and assignees are set
- Related issues are linked
- Ready for development
